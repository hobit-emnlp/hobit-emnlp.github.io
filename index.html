Why hoBIT?

Academic advising is profile-dependent: semantic similarity alone can surface plausible but inapplicable evidence.

Profile-blind RAG misses context

Graduation, course, and major rules vary by department, admission cohort, and major type, so the same query can require different documents.

Profile-aware indexing structures evidence

Each chunk is annotated with applicable profile values and the fields required to interpret it reliably.

Adaptive profiling keeps interaction light

proFILL acquires only the attributes needed for the current query, then updates the session profile on demand.

Same Question, Different Correct Answer

Semantically similar curriculum documents can apply to different students. proFILL makes evidence validity profile-aware.

Methodology

proFILL combines offline profile-aware indexing with online on-demand profiling for profile-dependent academic advising.

1

Intent Routing and Session-Profile Initialization

At query time, hoBIT classifies each incoming query into retrieval, ability, FAQ, greeting, or small talk. Only retrieval queries proceed to retrieval and answer generation, while direct intents are handled by templates or the FAQ store.

greetingFAQabilitysmall talkretrieval
For retrieval queries, proFILL identifies required profile attributes and initializes a time-limited session profile.
2

Offline Profile-Aware Indexing

Institutional materials are cleaned, chunked, and annotated with structured profile records before indexing. Non-null profile fields encode which students each chunk applies to.

sourceschunksprofile recordsQdrant
The default schema covers department, major type, grade, admission year, and student status.
3

On-Demand Adaptive Profiling and RAG

proFILL acquires missing information through query-driven profiling before retrieval and evidence-driven profiling when retrieved evidence reveals unresolved applicability requirements.

Profile values are incorporated through soft query augmentation and hard filtering over profile annotations.

- Official English score: one of TOEIC 650, TOEFL PBT 530/CBT 197/IBT 74, TOSEL(A) 512, TEPS 571, or IELTS 5.5

Dataset Construction

The evaluation data covers profile-grounded QA, open-ended advising, intent classification, and profile-annotated corpus chunks.

Track 1

Profile-Grounded QA

Each query is paired with a synthetic student profile, a manually encoded gold answer, a gold source, and a phrasing type.

Track 2

Open-Ended Advising

Questions without deterministic reference answers are evaluated by LLM judges for retrieved evidence quality and answer completeness.

Track 3

Intent Classification

Samples cover greeting, ability, FAQ, smalltalk, and retrieval so that only evidence-seeking queries enter the RAG pipeline.

Index

Profile-Annotated Corpus

Offline indexing annotates chunks over department, major type, grade, admission year, and student status.

Results

proFILL outperforms profile-blind RAG baselines in both retrieval and grounded answer generation.

Deployment MRR 0.593

proFILL achieves the strongest retrieval performance in the realistic setting where profiles must be acquired during interaction.

Source Match 0.749

Generated answers use the correct institutional sources more reliably than profile-blind baselines.

Grounded Correctness 0.625

Answers are evaluated for both correctness and grounding in appropriate sources.

Open-weight deployment competitive

Open-weight models remain viable for cost-effective on-premise deployment.
