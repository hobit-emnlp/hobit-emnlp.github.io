Abstract

University academic advising often depends on compact profile fields such as department, admission cohort, grade, and major type. A profile-blind RAG system can retrieve a semantically similar curriculum page while still returning the wrong answer for a specific student.

hoBIT is a deployed profile-aware RAG chatbot that enriches documents with profile metadata offline, acquires missing session profile fields through dual-asking, and injects the profile online as both a soft query prefix and a hard metadata filter. The system answers from source-tracked evidence while keeping the interaction session-based rather than login-based.

Across a 3,400-case, three-track evaluation, the paper finds that profile injection dominates other retrieval components, makes profile-blind reranking and HyDE redundant, works through a filter-prefix synergy, and remains viable with open-weight retrievers and generators for lower-cost on-premise deployment.

System Overview

hoBIT follows an offline-online design. Offline, curriculum PDFs, school regulations, notices, job posts, education data, and FAQ records are chunked, tagged with profile labels, and indexed into static and dynamic collections. Online, the system predicts which profile fields a query needs, asks only for missing fields, and injects the resulting session profile before retrieval.

Key Findings

  • Profile injection dominates. Source Match jumps from 0.070 with filtering alone to 0.585 with hoBIT, and Overall PASS rises from 6.8% to 47.4%.
  • Profile-blind reranking and HyDE become redundant. They help weak no-profile baselines at best, but underperform or hurt once profile injection is present.
  • Soft prefix and hard filter are complementary. The prefix improves recall by pulling the correct cohort into the candidate set, while the filter removes wrong-cohort distractors.
  • Open models are deployable. BGE-M3 and Qwen3 embeddings match or exceed the proprietary retriever baseline, while Qwen3-8B and EXAONE-7.8B remain competitive as generators.

Demonstration Flow

The demo highlights two user sessions. In a cold start, hoBIT asks for department and admission year proactively, then asks for major type reactively when cited evidence requires it. In a warm session, the same profile is reused silently, so the next advising question is answered immediately with citations.
